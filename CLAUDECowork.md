## 연구비내역서 작성 규칙

`서류 작업/` 폴더에서 연구비내역서를 작성할 때 아래 규칙을 따른다.

### 데이터 소스

거래명세서, 견적서, 카드전표 PDF 파일에서 품목명, 수량, 금액, 날짜 등을 추출하여 사용한다.

### 변경 대상 필드

| 필드 | 설명 |
|------|------|
| 사용일자 | PDF에서 추출한 결제일 |
| 금액 | 총 결제 금액. 반드시 `"315,000 원"` 형식으로 숫자 뒤에 ` 원`을 붙인다 |
| 사용내역 | `"{물건명} 외 {나머지 건수}건"` 형식. 물건이 1개면 물건명만 기재 |
| 날짜 | 문서 하단의 작성일을 결제일에 맞춰 수정 |

### 변경 금지 필드

아래 필드는 예시 문서의 값을 그대로 유지한다.

- 이름 (공동연구자, 연구책임자)
- 비고 (연구비 지출 항목)
- 연구개발과제와의 직접적인 관련성
- 과제번호, 지원기관, 소속, 연구기간

### 검증

작성 완료 후 원본 PDF(거래명세서, 견적서, 카드전표)와 대조하여 아래 항목이 일치하는지 확인한다.

- 사용일자가 카드전표의 결제일시와 일치하는지
- 금액이 카드전표의 합계 금액과 일치하는지
- 사용내역의 물건명이 거래명세서/견적서의 품목과 일치하는지
- 사용내역의 건수가 실제 품목 수와 맞는지

### 백업

검증 완료 후 해당 날짜 폴더(예: `서류 작업/0408`)를 아래 경로에 폴더째 복사한다.

```

```

## ToDo 워크플로우

모든 작업을 시작하기 전에 아래 워크플로우를 따른다.

### 1. ToDo.md 작성

작업 착수 전 워크스페이스 루트의 `ToDo.md` 파일에 새 항목을 **추가**한다. 기존 내용은 절대 삭제하지 않는다.

추가 형식:

```markdown
## YYYY-MM-DD | 작업 제목

- [ ] 세부 항목 1
- [ ] 세부 항목 2
- ...
```

### 2. 사용자 확인

`ToDo.md`에 항목을 추가한 뒤, 반드시 사용자에게 내용을 보여주고 승인을 받는다. 승인 전에는 실제 작업을 시작하지 않는다.

### 3. 작업 수행 및 체크

사용자 승인 후 작업을 진행하며, 각 항목 완료 시 체크박스를 `[x]`로 업데이트한다.

### 4. 기록 보관

`ToDo.md`는 누적 기록용이므로 완료된 항목도 삭제하지 않는다. 새 작업은 항상 파일 하단에 추가한다.

## 메일 회신 규칙

Outlook 메일에 대한 회신을 작성할 때 아래 규칙을 따른다.

### 템플릿

워크스페이스 루트의 `DocumentMailReply.md` 파일을 회신 템플릿으로 사용한다.

### 치환 규칙

템플릿의 `{보낸사람의 친근한 이름}` 플레이스홀더를 보낸 사람의 이름(성 제외)으로 치환한다.

- 예: 이진우 → 진우, 박찬영 → 찬영, 김민지 → 민지

### 회신 작성 절차

1. 크롬 브라우저에서 Outlook 메일함(https://outlook.cloud.microsoft/mail/)에 접속한다.
2. 대상 메일을 열고 보낸 사람, 참조, 첨부 파일 등 메일 정보를 확인한다.
3. "회신" 버튼을 클릭하여 회신 창을 연다.
4. 템플릿(`DocumentMailReply.md`)을 읽고, 플레이스홀더를 치환한 본문을 입력한다.
5. 원본 메일에 참조(CC)가 있었다면, 회신에도 동일하게 참조를 추가한다.
6. **보내기 전에 반드시 사용자에게 초안을 보여주고 승인을 받는다.** 승인 없이 메일을 발송하지 않는다.
7. 사용자 승인 후 "보내기" 버튼을 클릭하여 메일을 발송한다.
